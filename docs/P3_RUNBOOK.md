# P3 上線能力 Runbook（稽核 / 備份還原 / 效能基線）

## 1. 稽核覆蓋檢查

目標：確認關鍵操作都有 audit log，可追溯 who/when/what。

### 核心操作清單

- 欄位管理：建立、修改、刪除、分組維護（欄位模板已移除）
- 匯入任務：建立、重試
- 欄位合併任務：建立、完成/失敗
- 備份還原：建立快照、執行還原

### 驗證步驟

1. 依序執行上述操作各一次。
2. 呼叫 `GET /api/audit-logs?limit=100`。
3. 確認 `action/targetType/targetId/detail/at` 內容完整。

## 2. 備份還原演練（最小可用）

目標：可在測試環境完成「建立備份→資料變更→還原」閉環。

### 演練流程

1. 建立備份快照（透過現有 recovery API 或 UI）。
2. 執行一組資料變更（匯入、刪除、欄位合併）。
3. 執行還原到指定快照。
4. 抽樣比對：
   - 客戶總筆數
   - 核心欄位（`country/phone/provider/name`）
   - 欄位定義數與可見欄位

### 風險備註

- 還原前先公告，避免多人同時寫入造成觀察誤差。
- 還原後建議重新載入任務中心與主表，確保快取一致。

## 3. 效能基線驗證

目標：建立「可重跑」的查詢與任務耗時基線。

### 基線項目

- 客戶列表查詢（無篩選 / 常用篩選 / 含排序）
- 匯入任務（小檔、中檔、大檔）
- 欄位合併任務（10k/50k/100k 資料列）

### 測量輸出

- API P50/P95（ms）
- 任務總耗時（s）
- 失敗率與可重試率

### 實作提示

- 先套用 `server/prisma/perf-indexes.sql`
- 固定測試資料集（建議每次演練前 `db:seed:reset-fields`）
