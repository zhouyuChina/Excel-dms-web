# 內網備份還原演練 Runbook

## 目標
- 驗證快照可下載、可上傳、可還原。
- 將還原時間控制在可接受範圍（建議 P95 < 15 分鐘，依資料量調整）。
- 確認還原後資料完整性與查詢可用性。

## 演練頻率
- 正式環境：每月 1 次。
- 重大版本上線前：額外 1 次。

## 演練流程
1. 建立新快照（UI: 備份還原頁）。
2. 下載該快照並保存到離線儲存。
3. 刪除測試資料（小樣本）。
4. 使用「預覽還原」確認影響筆數。
5. 執行還原（full 或 selected）。
6. 核對資料筆數與抽樣欄位。
7. 將演練結果登記到維運日誌。

## 驗證指標
- `snapshot_create_success_rate >= 99%`
- `snapshot_restore_success_rate >= 99%`
- `restore_row_match_rate = 100%`
- `restore_p95_duration_minutes` 持續下降或持平

## 失敗處置
- 還原失敗：停止寫入流量、切換到前一個可用快照。
- 快照檔毀損：使用離線備份檔重建快照。
- 指標不達標：調整 chunk 大小與資料庫資源上限後重測。
